# build_tags_tx

Генератор XDR транзакций Stellar для массового создания операций `manage_data` (тегирования адресов).

## Описание

Скрипт создает одну транзакцию Stellar Public Network, содержащую N2 × N3 операций `manage_data`. Каждая операция устанавливает тег вида `D<i>_<j>` со значением в виде HEX-кодированного Stellar-адреса.

## Структура проекта

```
build_tags_tx/
├── config/             # Конфигурационные файлы
│   ├── p2.txt          # Адреса источников (operation source accounts)
│   ├── p3.txt          # Целевые адреса (data values)
│   └── config.env      # Параметры сети и комиссии
├── src/                # Исходный код
│   ├── build_tags_tx.zsh # Основной скрипт
│   └── utils.zsh       # Вспомогательные функции
├── out/                # Выходные файлы
│   ├── tx.xdr          # XDR транзакции
│   ├── tx.debug.json   # Декодированная транзакция
│   └── logs/           # Логи выполнения
├── tests/              # Тестирование
│   ├── test_runner.zsh # Запуск тестов
│   └── fixtures/       # Тестовые данные
└── README.md           # Эта документация
```

## Использование

1. Настройте файлы в `config/`:
   - `p2.txt`: по одному Stellar-адресу на строку (источники операций)
   - `p3.txt`: по одному Stellar-адресу на строку (значения тегов)
   - `config.env`: параметры сети (RPC_URL, NETWORK_PASSPHRASE, BASE_FEE)

2. Запустите скрипт:
   ```zsh
   zsh src/build_tags_tx.zsh
   ```

3. Результаты в `out/`:
   - `tx.xdr`: base64 XDR для подписи и отправки
   - `tx.debug.json`: JSON для проверки
   - `logs/run_*.log`: лог выполнения

## Тестирование

Запустите тесты:
```zsh
zsh tests/test_runner.zsh
```

Тесты включают:
- Smoke test (3×9 = 27 операций)
- Валидация адресов
- Обработка дубликатов
- Проверка расчета комиссии

## Зависимости

- `zsh`
- `stellar-cli` версии 23.4.1
- `xxd` (для HEX-кодирования)
- `jq` (для проверки JSON)

## Параметры

Скрипт использует параметры из `config.env`:
- `RPC_URL`: URL RPC-узла (по умолчанию https://stellar-soroban-public.nodies.app)
- `NETWORK_PASSPHRASE`: passphrase сети
- `BASE_FEE`: базовая комиссия за операцию (stroops)
- `MAX_FEE`: максимальная комиссия (опционально)

## Формат тегов

Тег: `D<i>_<j>`
- `i`: индекс источника (1..N2)
- `j`: индекс цели (1..N3)

Значение: HEX-кодированный Stellar-адрес цели.

## Пример

Для p2.txt с 2 адресами и p3.txt с 8 адресами:
- Создается транзакция с 16 операциями
- Каждая операция имеет уникальный тег D1_1, D1_2, ..., D2_8
- Значения - HEX-адреса из p3.txt

## Логирование

Каждый запуск создает лог в `out/logs/run_YYYYMMDD_HHMMSS.log` с деталями выполнения, включая:
- Количество операций
- Расчет комиссии
- Проверки корректности

## Безопасность

- Скрипт не отправляет транзакцию в сеть - только генерирует XDR
- Требуется отдельная подпись и отправка через stellar-cli или другие инструменты
- Проверяет валидность адресов перед генерацией
