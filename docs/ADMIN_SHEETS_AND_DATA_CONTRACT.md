# Структура листов и контракты данных

Документ описывает каждый лист таблицы: его назначение, режим работы, читателей/писателей и риски при изменении.

## Общие принципы работы с листами

### Fixed-column assumptions vs Header-based logic

В коде используются **два подхода** к работе с колонками:

#### Fixed-column assumptions (опасно)

Код **жёстко завязан** на номера колонок. Изменение порядка колонок **ломает** функционал:

| Функция | Колонки | Риск |
|---------|---------|------|
| [`getExistingTransferKeys()`](clasp/Резиденты%20Мабиз.js:93) | J (tx_hash), O (op_id) | Перестановка → потеря дедупликации |
| [`parseResidentsSheet()`](clasp/Резиденты%20Мабиз.js:731) | B (label), Q (Account_s), R (Asset_issuer) | Перестановка → неправильный маппинг |
| [`syncClickUpTasks()`](clasp/Резиденты%20Мабиз.js:1754) | G (updated_at) | Перестановка → сбой инкрементальной синхронизации |

#### Header-based logic (рекомендуется)

Код ищет колонки **по имени заголовка**. Безопаснее для изменений:

| Функция | Ищем | Безопасность |
|---------|------|--------------|
| [`buildFactMonthly()`](clasp/Резиденты%20Мабиз.js:2276) | `datetime`, `amount`, `project_id`, `class` | + |
| [`buildKpiRaw()`](clasp/Резиденты%20Мабиз.js:2406) | `Curator`, `Next_Action_Due` | + |
| [`updateResidentsFromTasks()`](clasp/Резиденты%20Мабиз.js:1873) | `Curator`, `Case_Status`, `Next_Action` | + |

**Правило**: При добавлении новых колонок — добавляйте **справа** от существующих. Не меняйте порядок фиксированных колонок.

### Операции с диапазонами

| Операция | Функция | Когда используется |
|----------|---------|-------------------|
| `getDataRange().getValues()` | [`parseConstSheet()`](clasp/Резиденты%20Мабиз.js:700), [`parseResidentsSheet()`](clasp/Резиденты%20Мабиз.js:731) | Чтение всего листа для анализа |
| `setValues()` массово | [`appendNewRows()`](clasp/Резиденты%20Мабиз.js:138), [`buildFactMonthly()`](clasp/Резиденты%20Мабиз.js:2276) | Запись множества строк |
| `clearContent()` | [`syncTransfersMemos()`](clasp/Резиденты%20Мабиз.js:621), [`buildFactMonthly()`](clasp/Резиденты%20Мабиз.js:2276) | Очистка перед перезаписью |
| `appendRow()` | Инициализаторы листов | Добавление одной строки |
| `getRange(row, col).getValue()` | [`syncClickUpTasks()`](clasp/Резиденты%20Мабиз.js:1754) | Точечное чтение (курсор) |

## Документация по листам

### CONST

| Параметр | Значение |
|----------|----------|
| **Назначение** | Конфигурация ключ-значение |
| **Создаёт** | Администратор вручную |
| **Читает** | [`parseConstSheet()`](clasp/Резиденты%20Мабиз.js:700) |
| **Пишет** | Администратор вручную |
| **Режим** | Reference (не изменяется скриптом) |
| **Ручное редактирование** | ✅ Да, ключи и значения |
| **Критичные заголовки** | A (ключ), B (значение) |

**Риски при изменении структуры**:
- Удаление/перестановка колонок A-B → скрипт не может прочитать конфигурацию
- Изменение имён ключей → функции не находят нужные настройки

### RESIDENTS

| Параметр | Значение |
|----------|----------|
| **Назначение** | Список резидентов с проектами, кураторами, управленческими полями |
| **Создаёт** | Администратор вручную |
| **Читает** | [`parseResidentsSheet()`](clasp/Резиденты%20Мабиз.js:731), [`buildKpiRaw()`](clasp/Резиденты%20Мабиз.js:2406), [`updateResidentsFromTasks()`](clasp/Резиденты%20Мабиз.js:1873) |
| **Пишет** | [`updateResidentsFromTasks()`](clasp/Резиденты%20Мабиз.js:1873) |
| **Режим** | Snapshot (обновляется точечно) |
| **Ручное редактирование** | ✅ Да, но аккуратно |
| **Критичные заголовки** | B (label), Q (Account_s), R (Asset_issuer), Curator, Next_Action_Due |

**Риски при изменении структуры**:
- Перестановка колонок B, Q, R → [`parseResidentsSheet()`](clasp/Резиденты%20Мабиз.js:731) возвращает неправильный маппинг
- Удаление `Curator`, `Next_Action_Due` → [`buildKpiRaw()`](clasp/Резиденты%20Мабиз.js:2406) не считает KPI
- Удаление `Work_Status/Notes` → [`updateResidentsFromTasks()`](clasp/Резиденты%20Мабиз.js:1873) не обновляет поле

### TRANSFERS

| Параметр | Значение |
|----------|----------|
| **Назначение** | Синхронизированные переводы Stellar |
| **Создаёт** | [`syncStellarTransfers()`](clasp/Резиденты%20Мабиз.js:149) |
| **Читает** | [`getExistingTransferKeys()`](clasp/Резиденты%20Мабиз.js:93), [`syncTransfersMemos()`](clasp/Резиденты%20Мабиз.js:621), [`buildFactMonthly()`](clasp/Резиденты%20Мабиз.js:2276), [`reclassifyTransfers()`](clasp/Резиденты%20Мабиз.js:1007) |
| **Пишет** | [`appendNewRows()`](clasp/Резиденты%20Мабиз.js:138), [`syncTransfersMemos()`](clasp/Резиденты%20Мабиз.js:621) |
| **Режим** | Append-only (только добавление новых строк) |
| **Ручное редактирование** | ⚠️ Только class_override, memo |
| **Критичные заголовки** | J (tx_hash), O (op_id), I (memo), direction, counterparty_type, project_id, class, class_override |

**Риски при изменении структуры**:
- Перестановка колонок J, O → [`getExistingTransferKeys()`](clasp/Резиденты%20Мабиз.js:93) теряет дедупликацию, дублируются транзакции
- Перестановка колонок I, J → [`syncTransfersMemos()`](clasp/Резиденты%20Мабиз.js:621) обновляет не те колонки
- Удаление `class_override` → невозможно ручное переопределение классификации
- Удаление `direction`, `counterparty_type` → [`classifyTransfer_()`](clasp/Резиденты%20Мабиз.js:959) не работает

## Матрица «лист → читатели → писатели → риски»

| Лист | Читатели | Писатели | Риски |
|------|----------|----------|-------|
| CONST | [`parseConstSheet()`](clasp/Резиденты%20Мабиз.js:700) | Администратор | Неправильные ключи → все функции падают |
| RESIDENTS | [`parseResidentsSheet()`](clasp/Резиденты%20Мабиз.js:731), [`buildKpiRaw()`](clasp/Резиденты%20Мабиз.js:2406), [`updateResidentsFromTasks()`](clasp/Резиденты%20Мабиз.js:1873) | Администратор, [`updateResidentsFromTasks()`](clasp/Резиденты%20Мабиз.js:1873) | Перестановка B/Q/R → неправильный маппинг |
| TRANSFERS | [`getExistingTransferKeys()`](clasp/Резиденты%20Мабиз.js:93), [`syncTransfersMemos()`](clasp/Резиденты%20Мабиз.js:621), [`buildFactMonthly()`](clasp/Резиденты%20Мабиз.js:2276) | [`appendNewRows()`](clasp/Резиденты%20Мабиз.js:138), [`syncTransfersMemos()`](clasp/Резиденты%20Мабиз.js:621) | Перестановка J/O → дубли, перестановка I/J → memo не обновляются |
| TRANSFERS_MEMO_QUEUE | [`syncTransfersMemos()`](clasp/Резиденты%20Мабиз.js:621) | [`syncStellarTransfers()`](clasp/Резиденты%20Мабиз.js:149), [`syncTransfersMemos()`](clasp/Резиденты%20Мабиз.js:621) | Ручное изменение → сбой очереди |
| ACCOUNTS | [`parseAccountsSheet()`](clasp/Резиденты%20Мабиз.js:765) | Администратор | Нет критичных рисков |
| PROJECT_MAP | [`parseProjectMapSheet()`](clasp/Резиденты%20Мабиз.js:776), [`mapProjectIdForTransfer_()`](clasp/Резиденты%20Мабиз.js:892) | Администратор | Удаление колонок → не работает маппинг |
