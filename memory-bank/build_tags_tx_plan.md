# План реализации подпроекта `build_tags_tx`

Этот документ описывает структуру и план реализации инструмента для генерации XDR транзакций Stellar с операциями `manage_data`.

## Структура каталогов

Подпроект будет располагаться в директории `build_tags_tx/` и иметь следующую структуру:

```text
build_tags_tx/
├── config/             # Конфигурационные файлы
│   ├── p2.txt          # Список адресов-источников (operation source accounts)
│   ├── p3.txt          # Список целевых адресов (data values)
│   └── config.env      # Переменные окружения (RPC_URL, NETWORK, FEE)
├── src/                # Исходный код
│   ├── build_tags_tx.zsh # Основной скрипт генерации
│   └── utils.zsh       # Вспомогательные функции (hex кодирование, валидация)
├── out/                # Выходные файлы (игнорируются git, кроме .gitkeep)
│   ├── tx.xdr          # Итоговый XDR в base64
│   ├── tx.debug.json   # Декодированная транзакция для проверки
│   └── logs/           # Логи выполнения (run_YYYYMMDD_HHMMSS.log)
├── tests/              # Тестирование
│   ├── test_runner.zsh # Скрипт для запуска тестов
│   └── fixtures/       # Тестовые данные (валидные и невалидные p2/p3)
└── README.md           # Инструкция по использованию
```

## Компоненты и логика

### 1. Основной скрипт (`src/build_tags_tx.zsh`)
- Чтение параметров командной строки и `config.env`.
- Валидация входных файлов `p2.txt` и `p3.txt`.
- Расчет комиссии (FEE).
- Итеративная сборка транзакции через `stellar-cli 23.4.1`.
- Вызов `stellar tx update seq-num next`.
- Генерация отчетов и логов.

### 2. Утилиты (`src/utils.zsh`)
- `to_hex_ascii()`: Конвертация Stellar-адреса в HEX для `manage_data`.
- `validate_address()`: Проверка формата G... адреса.
- `log_message()`: Форматированный вывод в консоль и файл лога.

### 3. Тестирование (`tests/test_runner.zsh`)
- **Smoke test**: Проверка создания транзакции на малом наборе данных.
- **Validation test**: Проверка реакции на некорректные адреса.
- **Integrity test**: Декодирование полученного XDR и сверка количества операций и адресов.

## Зависимости
- `zsh`
- `stellar-cli` версии **23.4.1**
- Доступ к Stellar RPC (для получения sequence number)
