## Паттерны кодирования

[2026-04-22 14:25:07] - **Snapshot sheet builders**: Отдельные builder-функции формируют конкретные RT read-model листы из общего foundation dataset, а orchestration-слой только задает порядок их выполнения.
[2026-04-22 14:25:07] - **Bootstrap и инициализация новых листов**: Перед записью RT snapshot-витрин выполняется создание или доведение листов до ожидаемого контракта, чтобы запись оставалась идемпотентной для новых и существующих таблиц.
[2026-04-22 14:25:07] - **Header-contract validation**: Любая запись в snapshot/read-model листы опирается на явную проверку заголовков и структуры листа вместо молчаливых допущений о колонках.
2026-01-14 18:50:56 - Обновление паттернов кодирования на основе анализа кода проекта Stellar Scrypt.
* **Структура функций**: Функции начинаются с комментариев и описания назначения. Внутренние хелперы имеют суффикс `_` (например, `loadConsts_`, `getOrCreateSheet_`).

* **API взаимодействия**: Работа с Google Sheets API (`SpreadsheetApp`), UrlFetchApp для HTTP запросов, PropertiesService для хранения курсоров.

## Архитектурные паттерны

[2026-04-22 14:25:07] - **Shared Domain Core + GAS bridges**: Общая доменная логика и foundation-построители остаются переиспользуемым core, а Google Apps Script entrypoints и menu actions выступают bridge-слоем для orchestration и интеграции с таблицей.
[2026-04-22 14:25:07] - **Resident-tracking как snapshot/read-model pipeline**: Resident-tracking строится как последовательность upstream sync и downstream builders, а не как набор независимых ad hoc отчетов.
[2026-04-22 14:25:07] - **Separation of upstream dataset and downstream views**: Foundation dataset и account-metadata intelligence рассматриваются как источники данных, а RT timeline, token flows и issuer structure — как производные read-model представления.
[2026-04-01 13:46:00] - **Слоистая архитектура (Layered Architecture)**: Разделение на Domain (чистая логика), Adapters (Sheets/Horizon) и Orchestration (Use Cases).
[2026-04-01 13:46:00] - **Repository Pattern (Header-based Discovery)**: Доступ к данным Sheets только через поиск колонок по заголовкам, а не по фиксированным индексам.

2026-01-14 18:50:56 - Определение архитектурных паттернов проекта Stellar Scrypt.
* **Модульная архитектура**: Код разделен на модули по функциональности (синхронизация активов, переводов, история активов). Каждая функция отвечает за конкретную задачу.
* **Пагинация с курсорами**: Использование курсоров Stellar Horizon API для обработки больших объемов данных. Курсоры сохраняются в PropertiesService для продолжения синхронизации.
* **Фильтрация данных**: Многоуровневая фильтрация: по типам операций (только payments), по участникам (фонды и резиденты), по активам (разрешенные активы из списка резидентов).
* **Отладка и аудит**: Создание отдельных листов для отладки (ASSETS_DEBUG, TRANSFERS_DEBUG) и ошибок (ASSETS_ERRORS). Включение сырых данных для анализа.
* **Интеграция с Google Sheets**: Динамическое создание и обновление листов. Использование `getRange().setValues()` для массовой записи данных.
* **Кросс-платформенность**: Код работает в среде Google Apps Script, совместим с различными ОС через абстракцию API.

## Паттерны тестирования

[2026-04-22 14:25:07] - **Operational flow verification**: Корректность RT-витрин проверяется через полный сценарий upstream sync → bootstrap/validation → downstream rebuild, а не через изолированный запуск отдельных view builders.
2026-01-14 18:50:56 - Идентификация паттернов тестирования в проекте.
* **Отладочные функции**: Специальные функции для точечной отладки (например, `debugSingleAccountAssets`), позволяющие тестировать отдельные аккаунты.
* **Логирование**: Запись ошибок и отладочной информации в отдельные листы для анализа. Использование `console.log` для промежуточных результатов.
* **Валидация данных**: Проверка наличия обязательных полей (например, `mustGet_` для констант), обработка пустых или некорректных ответов API, валидация пользовательского ввода (адреса Stellar, даты, активы).
* **Ручное тестирование**: Функции для ручного запуска и проверки результатов в Google Sheets (меню Stellar с пунктами обновления).
* **Защита от бесконечных циклов**: Лимиты на количество страниц (например, 30 страниц) для предотвращения зависаний при тестировании.
